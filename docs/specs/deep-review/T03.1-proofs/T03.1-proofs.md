### Key Design Decisions

- Follows `post_review.py` patterns exactly: `die()`, `warn()`, stdlib-only, argparse
- Accepts both bare array and `{"findings": [...]}` envelope for input flexibility
- Security dimensions use `security_min_confidence` (separate from general threshold)
- Pipeline order: threshold → exclusions → injection → disagreement → tag
- `detect_disagreement()` adds `consensus_count`, `consensus_boost`, `contradiction` fields
- `tag_findings()` uses severity (critical/high always main) then dimension to route findings
