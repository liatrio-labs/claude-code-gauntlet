## Implementation

### 1. Anti-anchoring note in step 1 (RC-5)
Added **Anti-anchoring** clause to the "Trace the intent first" step: instructs the agent
to analyze paths NOT emphasized by the change summary, specifically read/retrieval paths
when write/invalidation is described, proxy pass-through methods, and non-happy auth paths.

### 2. New step 10: Delegation/proxy/wrapper tracing (RC-1)
New investigation step explicitly requiring the agent to verify:
- (a) wrapper delegates to underlying implementation, not back to itself
- (b) wrapper uses the correct delegate reference (delegate vs this vs session)
- (c) every pass-through method calls the underlying layer

Concrete example provided: "a cache calling `this.method()` instead of `delegate.method()`
loops forever or returns stale data without error"

### 3. New step 11: Auth-context null-handling (V4-06)
New investigation step for authentication/authorization code requiring the agent to verify
null guards for user/member/organization objects on all auth paths, with specific coverage
of: API key paths (member=None), service account paths (user attributes may be missing),
and short-circuit OR expressions that only skip the right side when left is True.

### 4. Auth-context bullet in Null/undefined handling section
Added auth-specific pattern to the "Null/undefined handling" bullet list as a quick-reference
signal when scanning code.

### 5. Test code logic bug scope clarification (RC-2)
In "What you do NOT report": Changed "Error handling in test code" to "Error handling style
in test code" and added a Note clarifying that logic bugs in test code (wrong assertions,
incorrect fixtures, mismatched cleanup) ARE in scope when the test file is part of the PR.

### 6. phase2-triage.md 2e: Delegation patterns as high-risk (RC-1)
Added "files implementing a cache, proxy, decorator, or delegation pattern" to the high-risk
classification in step 2e. High-risk files receive expanded context (callers, callees,
related tests) ensuring the bug-detector gets the full class hierarchy needed for
delegation tracing.
