## Implementation Summary

The `verify_factual()` function in `scripts/verify_findings.py` was implemented with:

1. **File content reading** (R02.3.1): Reads lines `[line_start, line_end]` using Python file I/O with
   UTF-8 encoding and graceful binary file detection. Returns `False` if file not found or lines out of range.

2. **Symbol verification via grep** (R02.3.2): Extracts symbol names from finding description/evidence
   using regex patterns (backtick spans, quoted names, CamelCase, snake_case function calls).
   Runs `grep -rn --include=*.py -l <symbol> .` for each symbol not already present in the relevant lines.

3. **confidence=0 for wrong facts** (R02.3.3): Sets `finding["confidence"] = 0` in three cases:
   - File does not exist on disk (also returns `False` to eliminate)
   - line_start/line_end out of range (also returns `False` to eliminate)
   - Referenced symbols not found in codebase (returns `True` to degrade but keep)

4. **Verification metadata**: Sets `finding["factual_verification"]` dict with `verified`, `reason`,
   and `code_at_lines` fields on every finding processed.

5. **Edge cases handled**:
   - No `line_start` → skip verification, return `True`
   - Binary files → skip verification, return `True` with warning
   - OSError reading file → return `False`, confidence=0
