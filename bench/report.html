Every delivered comment lands in one bucket

  • golden-matched — matched a known-good finding from the hand-labelled golden set
  • valid-extra — a real issue that just isn't in the golden set — never counted against us
  • noise — wrong, or not worth a reviewer's time

The metrics

  • Recall = goldens caught ÷ all goldens. The headline, gated metric.
  • Noise rate = noise ÷ all delivered comments — ‘how often are we wrong’. Gated: must stay under the ceiling.
  • Valid-extra = real issues beyond the golden set. Reported, never penalised.
  • Precision † counts valid-extras as misses, so it sinks as volume grows. Reported, not gated — read noise rate instead.

The three bars that define success

  • 62.7% — top-anchor recall (best external tool) — the bar a run must beat
  • 49.1% — deep-review v2 recall — the prior baseline
  • 24.0% — noise ceiling — a run above it fails the gate

How much a run is trusted

  • Smoke — 3 PRs / 4 goldens — directional only
  • Subset — 15 PRs / 59 goldens — gate-grade
  • Holdout — 10 fresh PRs — confirmation

Smoke numbers are directional and never pass or fail a gate; a subset run is gate-grade; a holdout run confirms it on fresh PRs. By owner decision v3 delivers ~4× the comments of v2, so its lower precision is a bigger denominator, not weaker findings.
